# Phase 2: 思路生成与筛选（Idea Creator）

研究主题：**$ARGUMENTS**

## 步骤

### Phase 0: 加载文献上下文

1. 读取 `idea-stage/LITERATURE_SURVEY.md`（Phase 1 输出）
2. 读取 `idea-stage/REF_PAPER_SUMMARY.md`（如有参考论文）
3. 如无文献调研，用 WebSearch 快速获取上下文

### Phase 1: 头脑风暴（发散）

用 WorkBuddy 配置的模型生成 8-12 个具体研究思路。对每个思路要求：

1. 一句话摘要
2. 核心假设（预期发现什么，为什么）
3. 最小可行实验（最便宜的验证方式）
4. 贡献类型：实证发现 / 新方法 / 理论结果 / 诊断工具
5. 风险等级：LOW / MEDIUM / HIGH
6. 预估工作量：天 / 周 / 月

优先满足：
- 可用中等算力测试（8x RTX 3090 或更少）
- 无论正负结果都有意义（都能发表）
- 不是简单的"把 X 应用到 Y"
- 与已知近期工作有明确区分

### Phase 2: 第一轮筛选

对每个思路快速评估：

1. **可行性**：算力需求 / 数据可用性 / 实现复杂度。跳过 >1 周 GPU 的
2. **新颖性快检**：2-3 次 WebSearch 查该 idea 是否已存在
3. **影响力**："So what?" 测试——实验成功会改变认知吗？

通常 8-12 个缩减到 4-6 个。

### Phase 3: 深度验证（对 Top 思路）

1. **新颖性验证**：对每个 top idea 执行 Phase 3 的完整流程
2. **魔鬼代言人**：模型扮演批评者——最强反对意见 / 最可能失败模式 / 排名
3. 合并评估，选出 2-3 个进入 pilot

### Phase 4: Pilot 实验（可选，有 GPU 时）

1. 设计最小实验（单 seed / 小规模 / 30min-2h per idea）
2. 用 TaskCreate/TaskUpdate 追踪 GPU 时长
3. 超时自动终止，收集部分结果
4. 按实证信号重排排名

> 无 GPU 或纯理论方向时跳过，标记"需要 pilot 验证"。

### Phase 5: 输出

写入 `idea-stage/IDEA_REPORT.md`：

```markdown
# 研究思路报告

**方向**: [研究方向]
**生成**: [日期]
**评估**: X 生成 → Y 筛选后 → Z pilot → W 推荐

## 排序后的推荐思路

### 🏆 Idea 1: [标题] — 推荐
- 假设 / 最小实验 / 预期结果
- 新颖性: X/10 — 最近工作: [论文], 区别: [什么不同]
- 可行性 / 风险 / 贡献类型
- Pilot 结果 / 审稿人可能反对 / 为什么值得做

### Idea 2: [标题] — 备选
### Idea 3: [标题] — 值得探索

## 淘汰的思路
| # | Idea | 淘汰原因 |

## Pilot 实验结果
| Idea | GPU | 时间 | 关键指标 | 信号 |

## 建议执行顺序
## 下一步
```

## 规则

- 用户给的是方向，不是 idea——你的任务是生成 idea
- 先求量再求质——广泛头脑风暴，然后无情筛选
- 正面和负面结果都有发表价值
- 估计算力成本，>1000 GPU-hours 不可行
- "把 X 应用到 Y" 是最低级的 idea，追求更深层的问题
- 包含淘汰 idea——节省未来时间
- 方向太宽（如"NLP"、"计算机视觉"）时停止，要求用户缩小范围

## 检查点

用 AskUserQuestion 展示排序后的 ideas，选项：`["继续 top ideas", "换个约束重新生成", "只关注 Idea 2"]`
