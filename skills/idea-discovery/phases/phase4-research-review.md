# Phase 4: 模拟审稿（Research Review）

审稿对象：**$ARGUMENTS**

## 步骤

### Step 1: 汇集研究上下文

1. 读取项目文件：
   - `idea-stage/IDEA_REPORT.md` — 排序后的 ideas
   - `idea-stage/NOVELTY_CHECK_*.md` — 新颖性报告
   - `idea-stage/LITERATURE_SURVEY.md` — 文献全景
   - `idea-stage/REF_PAPER_SUMMARY.md` — 参考论文分析
2. 提取：核心主张/方法论/已有结果/已知弱点/目标会议

### Step 2: 初始审稿（Round 1）

模型扮演**高级 ML 审稿人**（NeurIPS/ICML 级别），覆盖：

#### 2.1 优点
- 核心贡献是什么？
- 为什么对社区重要？
- 方法最强方面？

#### 2.2 缺点
- 逻辑漏洞或未证实的声明
- 缺失的实验（审稿人会期望什么？）
- 基线比较（够不够？够公平吗？）
- 新颖性担忧
- 可扩展性/泛化性
- 可复现性

#### 2.3 给作者的问题
3-5 个具体、可回答的问题。

#### 2.4 评分
- 总分：X/10
- 置信度：High / Medium / Low
- 建议：Strong Accept / Accept / Weak Accept / Borderline / Weak Reject / Reject

#### 2.5 如何提升到 Accept
具体、可操作的改进建议。

### Step 3: 迭代改进（Rounds 2-N，最多 3 轮）

每轮：
1. 处理上一轮反馈（可修复/致命/修复最小努力）
2. 针对性追问
3. 产出具体交付物：实验设计/论文大纲/主张矩阵/消融优先级

### Step 4: 收敛

停止条件：
- 所有关键弱点已有具体方案
- 清晰的实验路线图
- 叙事结构已确定
- 或 MAX_REVIEW_ROUNDS = 3 到达

### Step 5: 输出

写入 `idea-stage/REVIEW_[idea_short_name].md`：

```markdown
# 研究审稿: [Idea 标题]

**审稿人**: WorkBuddy 配置模型（深度审稿）
**日期**: [today]
**轮次**: [N]

## Round 1: 初始审稿

### 摘要 / 优点 / 缺点 / 给作者的问题

### 评分
| 维度 | 分数 (1-10) | 备注 |
| 新颖性 | X | |
| 重要性 | X | |
| 可靠性 | X | |
| 清晰度 | X | |
| 总分 | X | |

### 建议 + 如何提升到 Accept

## Round 2: 改进
[更新后的评估]

## 最终共识
### 结论 / 必须做 / 最好做 / 最小实验包 / 主张矩阵
```

## 规则

- 残忍诚实 — 隐藏弱点导致更差反馈和浪费
- 第一轮发送完整上下文
- 推回你不同意的批评，但接受合理的
- 每个弱点都附带具体修复建议
- 审稿文档必须自包含
- 不要虚高分数 — 诚实的 6/10 比虚假的 9/10 更有用
- 考虑目标会议的门槛
