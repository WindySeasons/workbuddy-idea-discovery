# Phase 6: 实验规划（Experiment Plan）

规划对象：**$ARGUMENTS**

## 概览

当方法稳定到"下一步该跑什么实验来为论文辩护？"这个问题时使用。

目标：把方案变成 **主张 → 证据 → 运行顺序** 路线图。

## 常量

- OUTPUT_DIR = `refine-logs/`
- MAX_PRIMARY_CLAIMS = 2，MAX_CORE_BLOCKS = 5，MAX_BASELINE_FAMILIES = 3
- DEFAULT_SEEDS = 3

## 工作流

### Phase 0: 加载方案上下文

读取：
- `refine-logs/FINAL_PROPOSAL.md`
- `refine-logs/REVIEW_SUMMARY.md`
- `refine-logs/REFINEMENT_REPORT.md`
- `idea-stage/IDEA_REPORT.md`

提取：问题锚点 / 主导贡献 / 辅助贡献 / 关键审稿关切 / 数据算力约束 / 核心前沿组件

### Phase 1: 冻结论文主张

- **主主张**：主要机制级贡献
- **辅助主张**：可选，直接加强论文故事
- **需排除的反主张**：如"增益只来自更多参数"/"现代组件只是装饰"
- **最小令人信服的证据**：什么能让审稿人相信？

### Phase 2: 构建实验故事线

5 个默认 block（按需裁剪）：

1. **主锚点结果** — 方法是否解决了实际瓶颈？
2. **新颖性隔离** — 主导贡献本身是否重要？
3. **简洁/优雅检查** — 能否避免更大或更碎片化的版本？
4. **前沿必要性检查** — 核心现代组件确实是正确工具吗？
5. **失败分析/定性诊断** — 方法仍遗漏了什么？

每个 block 归类到：主论文 / 附录 / 裁剪。

优先选强基线家族而非多个弱基线。

### Phase 3: 完整规格化每个实验 block

对每个保留的 block 指定：

- 测试的主张
- 为什么存在
- 数据集/划分/任务
- 对比系统：最强基线 + 消融 + 变体（仅必要的）
- 指标：决定性指标优先，次要指标其次
- 设置细节：backbone / 冻结vs可训练 / 关键超参 / 训练预算 / seeds
- 成功标准
- 失败解读
- 目标表格/图

特殊规则：
- 简洁检查应对比最终方法 vs 过度构建变体
- 前沿必要性检查应对比选择的现代组件 vs 最强可替代的更简单方案

### Phase 4: 生成执行顺序

里程碑结构：

1. **Sanity 阶段** — 数据管线 / 指标准确性 / 快速过拟合或 toy split
2. **基线阶段** — 复现最强基线
3. **主方法阶段** — 在主设置上运行最终方法
4. **决策阶段** — 新颖性/简洁/前沿必要性的决定性消融
5. **完善阶段** — 鲁棒性 / 定性图 / 附录

每个里程碑估算：算力成本 / 预计时间 / stop-go 决策门 / 风险与缓解

区分 **必跑** 和 **最好有** 的实验。

### Phase 5: 输出

#### 5.1: `refine-logs/EXPERIMENT_PLAN.md`

结构：主张映射（主张→证据→关联block）/ 论文故事线 / 实验Blocks / 运行顺序与里程碑 / 算力与数据预算 / 风险与缓解 / 最终检查清单

#### 5.2: `refine-logs/EXPERIMENT_TRACKER.md`

```markdown
| Run ID | 里程碑 | 目的 | 系统/变体 | 划分 | 指标 | 优先级 | 状态 | 备注 |
|--------|--------|------|-----------|------|------|--------|------|------|
| R001   | M0     | sanity | ... | ... | ... | MUST | TODO | ... |
```

#### 5.3: 汇报

```
实验计划就绪。
必跑 block: [Block 1], [Block 2]
最高风险假设: [风险]
前 3 个运行: [run 1], [run 2], [run 3]
```

## 规则

- 每个实验必须为主张辩护，不改变审稿人信念的实验砍掉
- 优先紧凑的论文故事，先设计主表再加必要消融
- 显式辩护简洁性——包含删除研究或更强但臃肿的变体对比
- 显式辩护前沿选择——证明为什么现代组件比最强更简单替代好
- 优先强基线而非长基线列表
- 区分必跑和最好有
- 复用方案约束，不编造不现实的预算
- 不编造结果
- 大文件处理：Write 失败时用 Bash 分块写入
